Before You Start

  1. Open the org: Have your Salesforce org loaded and ready
  2. Find a household: Navigate to a Person Account (e.g., Henry Miller)
  3. Identify a visit: Have a Visit record in "Scheduled" status ready
  4. Mobile device: Have Salesforce Mobile app installed and logged in
  5. Demo dictation ready: Keep this page open in a tab

Step 1: Home Page Experience

1 Navigate to Home

Action: Click on "Home" from the app launcher or navigation

Show: Point out the My Day Component

  • Today's scheduled visits
  • Status summary (Scheduled, In Progress, Pending)
  • Quick actions: Start Visit, Get Directions
Talking Point: "This is my worker's personalized dashboard. I can see all my visits for today, their priority, and quickly navigate to households that need attention."

Step 2: Visit Record (Desktop)

2 Open a Visit Record

Action: Click on a visit from My Day or navigate to a Visit record

Show on desktop:

  • Path: Current status should be "Scheduled"
  • Household Profile Card: Shows last visit date, risk level, authorized hours
  • Visit fields: Planned time, address, visitor
  • Benefits eligibility section: Empty rankings fields
Path Status: Scheduled

Step 3: Mobile Experience - Field Visit

3A Open Visit on Mobile

Action: Open Salesforce Mobile app and navigate to the same Visit

Show: Mobile-optimized view of the visit record

Scenario: "I'm now at the household conducting the visit. I need to document my observations in real-time."
3B Update Path to "Visit In Progress"

Action: Click the Path and select "Visit In Progress"

Why: Shows that the worker has started the visit

Path Status: Visit In Progress
3C Add Visit Notes (Dictation)

Action: Scroll to Visit Notes Dictation field

Demo: Use voice-to-text or type example notes:

Client is ambulatory but requires assistance with meal preparation due to arthritis in hands. Home is clean and safe. Client expressed satisfaction with current care plan.
3D Demo Handwritten Note Scanner

Action: Click "Scan Handwritten Visit Notes" quick action

Show:

  • Camera interface opens
  • Point at handwritten notes (or use sample image)
  • AI transcribes handwriting to text
  • Transcribed text populates the Visit Notes field
Talking Point: "For workers who prefer pen and paper, they can simply photograph their handwritten notes and our AI will transcribe them automatically."
3E Save the Visit

Action: Tap Save

Result: Visit notes are captured, ready for processing when back at the desk

Step 4: Back to Desktop - Parse & Generate

4A Return to Desktop View

Action: Switch back to desktop browser with the Visit record open

Show: Visit notes are now populated from mobile entry

4B Update Path to "Documentation Pending"

Action: Update Path to "Documentation Pending"

Why: Visit is complete, now processing documentation

Path Status: Documentation Pending
4C Paste Full Demo Dictation

Action: Go to demo dictation page ↗

Copy: The full sample dictation text

Paste: Into the Visit Notes Dictation field (replacing or appending to existing notes)

Save: Click Save

Why this step: This gives us rich, realistic visit documentation to parse and generate rankings from.
4D Parse Text

Action: Click "Parse Visit Notes" quick action

Show:

  • AI extracts structured data from unstructured notes
  • Populates Meal Preparation, Ambulation, Domestic Services evidence fields
  • Populates rationale for each category

Result: Structured fields are now populated, ready for ranking generation

4E Generate Benefits Eligibility
Agentforce Action

Action: Click "Generate Benefits Eligibility" quick action

Show:

  • AI analyzes evidence and rationale
  • Generates proposed rankings (1-5 scale)
  • Calculates proposed hours based on IHSS guidelines
  • Flags exceptions if hours exceed thresholds

Fields populated:

  • Meal Preparation Proposed Ranking & Hours
  • Ambulation Proposed Ranking & Hours
  • Domestic Services Proposed Ranking & Hours
  • Exception flags (if applicable)

Step 5: Ask IHSS Employee Agent

5A Open Agent Panel

Action: Click the Agentforce icon in the utility bar (or Einstein icon)

Show: IHSS Employee Agent chat panel opens

5B Verify Exception Threshold
Agent Query

Scenario: "I see an exception flag on meal preparation. Let me verify the threshold."

Ask the agent:

What is the exception threshold for meal preparation hours?

Expected Response: Agent cites IHSS policy documentation showing the exact threshold (e.g., 12.5 hours/week)

Talking Point: "The agent has access to the complete IHSS policy library and can instantly cite regulations, so workers don't have to memorize complex rules."
5C Review and Adjust Rankings

Action: Review the proposed rankings

Adjust if needed: Update any rankings manually based on professional judgment

Update: Change proposed rankings to approved rankings

  • Meal Preparation Approved Ranking: Rank 3
  • Ambulation Approved Ranking: Rank 2
  • Domestic Services Approved Ranking: Rank 4

Step 6: Generate Narrative & Finalize

6A Generate Draft Narrative
Agentforce Action

Action: Click "Generate Draft Narrative" quick action

Show:

  • AI synthesizes all visit data into professional narrative
  • Includes rankings, hours, observations, and rationale
  • Formats as compliant IHSS documentation

Result: Draft Narrative field is populated with formatted HTML narrative

6B Review Draft Narrative

Action: Scroll to Draft Narrative field

Show: Formatted narrative with:

  • Visit summary
  • Service category assessments
  • Approved rankings and justifications
  • Worker attestation section
Talking Point: "What used to take 45 minutes of typing is now generated in seconds, and the worker just needs to review and approve."
6C Approve and Finalize Visit
Screen Flow

Action: Click "Approve and Finalize Visit" quick action

Screen 1 - Review and Attest:

  • Shows approved rankings summary
  • Displays draft narrative preview
  • Requires worker attestation checkbox: "I attest that this visit was conducted and documented accurately"
6D Input Risk Assessment

Screen 2 - Risk and Priority:

Input:

  • Overall Risk Level: Select Medium
  • Highest Need Category: Select Meal Preparation

Click: Finalize Visit

6E Confirm Submission

Result:

  • Visit status updates to Under Review
  • Narrative Status: Submitted
  • Report Status: Submitted
  • Worker Approval Date: Today's date
  • Final Narrative locked
Talking Point: "The visit is now in the supervisor's queue for review. The system automatically updated the household's Last Visit Date and all audit fields."

Step 7: Employee Agent - Workload Management

7A Check Backlog
Agent Query

Scenario: "Now that this visit is submitted, let me check what else needs my attention."

Ask the agent:

Show me my backlog

Expected Response: Agent queries visits assigned to user and shows:

  • Visits in "Documentation Pending" status
  • Overdue visits (>90 days)
  • Visits with exceptions under review
7B Find Incomplete Visits
Agent Query

Ask the agent:

Which visits are missing rankings?

Expected Response: Agent identifies visits where approved rankings are null

  • Lists visit numbers and households
  • Provides direct links to each visit
  • Shows how many fields are incomplete per visit
Talking Point: "The agent helps workers prioritize their workload and ensures no visits fall through the cracks."
7C Policy Questions
Additional Queries

Other questions you can demo:

Troubleshooting

Q: Handwritten note scanner isn't working

A: Ensure you've granted camera permissions to Salesforce Mobile. If unavailable, skip this step and focus on voice dictation.

Q: Agent isn't responding or gives generic answers

A: Check that the agent has access to the IHSS policy data library. Try rephrasing questions to be more specific.

Q: Benefits eligibility quick action fails

A: Verify that evidence and rationale fields are populated from the Parse Text step. Agent needs structured input.

Q: Path isn't updating

A: Make sure you're clicking the Path component at the top of the record, not just changing status fields manually.

Q: Draft narrative shows HTML tags

A: This is expected in edit mode. The Approve & Finalize flow displays it properly formatted.
